# Changes

In-flight changes. One folder per change at `context/changes/<created-date>-<change-id>/` (the date prefix makes a change's age visible from `ls` alone, same convention as `context/archive/`), identified by a `change.md` identity file whose `change_id` stays the bare slug. Created via `/10x-new`. Holds research, frame, plan, reviews, and other change-scoped artifacts.

When a change is complete, archive it with `/10x-archive` to move it under `context/archive/`.
